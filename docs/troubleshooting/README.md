# 트러블슈팅 문서 작성 가이드

이 디렉터리에는 프로젝트 진행 중 발생하는 문제와 해결 방법을 공유하기 위한 문서들을 저장합니다. 다른 팀원들이 비슷한 문제를 겪을 때 빠르고 효율적으로 해결할 수 있도록 돕는 것을 목표로 합니다.

## 문서 작성 방법

### 파일 이름

- 문제의 핵심 내용을 담아서 명확하고 간결하게 작성합니다.
- 영어 소문자와 하이픈(-)을 사용합니다.
- 예: `s3-image-upload-failed.md`, `database-connection-error.md`

### 문서 구조

모든 트러블슈팅 문서는 다음 구조를 따르는 것을 권장합니다.

```markdown
# 문제 제목 (예: S3 이미지 업로드 실패)

## 문제 상황 (Problem)

- 어떤 작업을 할 때 문제가 발생했는지 구체적으로 작성합니다.
- 발생한 에러 메시지, 로그, 스크린샷 등을 포함하면 좋습니다.
- 재현 가능한 절차를 설명해주면 가장 좋습니다.

## 원인 (Cause)

- 문제의 근본적인 원인을 분석하고 설명합니다.
- 여러 원인이 있을 경우 목록으로 나누어 작성합니다.

## 해결 방법 (Solution)

- 문제를 해결하기 위한 구체적인 절차를 순서대로 작성합니다.
- 필요한 명령어, 코드 수정 내용, 설정 변경 방법 등을 명확하게 기재합니다.
- 해결 방법을 적용한 후, 정상적으로 동작하는지 확인하는 방법도 함께 안내합니다.
```

## 템플릿

새로운 트러블슈팅 문서를 작성할 때 아래 템플릿을 복사하여 사용하세요.

```markdown
# [문제 요약]

## 문제 상황

(문제 발생 상황을 구체적으로 서술)

## 원인

(문제 발생 원인을 분석하여 서술)

## 해결 방법

(해결 방법을 순서대로 서술)
```
